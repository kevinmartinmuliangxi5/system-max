### 2. Confidence 计算

#### ❌ v3.0 (改进前)

| Loop | 实际工作 | 显示 Confidence | 问题 |
|------|----------|----------------|------|
| 1 | 读取 3 个文件 | 70% | ❌ 未反映读取操作 |
| 2 | 创建 10 个文件 | 70% | ❌ 大量工作却是 70% |
| 3 | 修改 1 个文件 | 70% | ❌ 小工作也是 70% |
| 4 | 只探索，无修改 | 70% | ❌ 无实质工作也是 70% |
| 5 | 任务完成 | 70% | ❌ 完成了还是 70% |

**计算逻辑**：
```bash
confidence = 0          # Claude 输出无此字段
confidence += 50        # 固定基础分
confidence += 20        # 有结构化响应
= 70% (永远是 70%)
```

**问题**：
- 😕 所有情况都显示 70%
- 😕 无法区分大小任务
- 😕 无法反映实际进展
- 😕 对用户没有参考价值

---

#### ✅ v3.1 (改进后)

| Loop | 实际工作 | 计算过程 | 显示 Confidence |
|------|----------|----------|----------------|
| 1 | 读取 3 个文件 | 30(基础) + 15(3工具) + 5(输出) = 50% | 50% ✅ |
| 2 | 创建 10 个文件 | 30(基础) + 40(10文件) + 24(8工具) + 15(输出) = 95% | 95% ✅ |
| 3 | 修改 1 个文件 | 30(基础) + 10(1文件) + 10(2工具) + 5(输出) = 55% | 55% ✅ |
| 4 | 只探索，无修改 | 30(基础) + 0(0文件) + 21(5工具) + 10(输出) = 61% | 61% ✅ |
| 5 | 任务完成 | 明确完成信号 | 100% ✅ |

**计算逻辑**：
```bash
# 基础分
confidence = 30                    # 结构化响应

# 文件修改分 (0-40分)
if 文件数 >= 10: confidence += 40
elif 文件数 >= 5: confidence += 30
elif 文件数 >= 3: confidence += 20
else: confidence += 文件数 × 10

# 工具调用分 (0-30分)
if 工具数 <= 3: confidence += 工具数 × 5
else: confidence += 15 + (工具数-3) × 3

# 输出长度分 (0-15分)
if 输出 > 2000: confidence += 15
elif 输出 > 500: confidence += 10
elif 输出 > 100: confidence += 5

# 完成时
if 有完成信号: confidence = 100

# 上下限
confidence = max(20, min(95, confidence))
```

**改进**：
- ✅ 准确反映工作量
- ✅ 区分大小任务
- ✅ 识别完成状态
- ✅ 对用户有实际参考价值

## 🎯 退出机制验证

### ❌ v3.0 - 用户担心
```
"Confidence 一直 70%，会不会影响退出？"
"如果 confidence 太低，会不会无限循环？"
```

### ✅ v3.1 - 明确说明
```
断路器判断依据：
  ✅ consecutive_no_progress  (连续无进展)
  ✅ exit_signal              (完成信号)
  ✅ consecutive_same_error   (连续错误)
  ❌ confidence               (不参与判断)

结论：
  ✅ confidence 改进完全不影响退出机制
  ✅ 退出机制只基于进展判断，与分数无关
  ✅ 可以放心使用新的 confidence 计算
```
