# AI 生图 App 企业级 UI 设计方案

## 1. 目标与边界
- 目标：基于《AI生图软件PRD（MVP）》输出两套可直接用于 Vibe Coding 的 `.pen` 高保真企业级 UI。
- 非目标：本次不实现代码、不做动效参数微调、不做品牌商标资产。
- 约束：覆盖 PRD 三大核心页面（对话主页面、全屏编辑器、结果页），并体现加载/错误/空态。
- 验收：每套方案包含完整信息架构、组件命名清晰、可在 Pencil MCP 中截图验证。

## 2. 两套方案（含取舍）

### 方案 A（推荐）：Enterprise Calm
- 关键词：稳健、可信、生产力。
- 视觉：浅色底 + 深蓝品牌色 + 青绿强调色，模块化卡片。
- 优势：企业场景接受度高，开发风险低，信息密度可控。
- 风险：视觉冲击力中等。

### 方案 B：Neo Tech Ops
- 关键词：未来感、专业工具、性能导向。
- 视觉：深色工作台 + 电蓝/青色高亮 + 半透明分层。
- 优势：工具属性强，适合突出“编辑器能力”。
- 风险：需要更严格对比度控制。

### 方案 C：Editorial Minimal
- 关键词：简洁、留白、内容导向。
- 结论：不采纳。对“企业级工具感”和复杂状态承载不足。

## 3. 信息架构与页面清单
每套均包含 3 屏：
1. `Chat Home`：顶部状态栏 + 对话流 + 底部输入区 + 快捷操作。
2. `Editor Fullscreen`：画布区 + 工具栏（涂抹/擦除/撤销/重做）+ 参数抽屉。
3. `Result`：大图预览 + 成本/耗时日志摘要 + 保存/继续编辑/重试。

## 4. 状态态设计
- 加载态：自定义脉冲条 + 进度文本。
- 错误态：错误原因 + 重试按钮 + 降级建议。
- 空态：首图引导卡 + 示例 Prompt Chips。

## 5. 可开发性约束
- 统一 8pt 栅格。
- 组件命名面向实现：`TopBar`, `MessageCard`, `PromptInput`, `ToolDock`, `ResultActionBar`。
- 保证按钮最小触控区 >= 44px。
- 所有文本层级保持 Type Scale（12/14/16/20/28）。

## 6. 交付物
- `ai-image-ui-enterprise-calm.pen`
- `ai-image-ui-neo-tech-ops.pen`

## 7. 评审结论
按“方案 A + 方案 B”并行产出，两套风格差异化，结构一致以便后续工程快速切换主题。
