## What We're Building

一款面向公务员考试考生的 **AI 面试训练系统**，基于《面试宝典》方法论，提供岗位深度定制化的面试辅导。

### 核心价值主张

1. **岗位画像调节器** - 三大岗位类型（执法/服务/综合）+ 刚性柔性滑块，解决"千人千面"问题
2. **双脑校验引擎** - 规则引擎（硬性检查步骤）+ 大模型（感性判断语气用词）
3. **评分溯源系统** - AI 引用考生原话，标注每句话的加分/扣分原因

### MVP 功能范围

- [x] **训练模式**（核心）：AI 生成题目 → 录音/文本作答 → 即时 AI 反馈（评分溯源、逻辑诊断）
- [x] **模拟考试**：
  - 规则：3 道题，每题 4 分钟思考 + 3 分钟作答
  - 题型组合：综合分析 + 应急应变 + 人际关系（随机抽取）
  - 全部作答完成后统一评分
- [ ] 能力画像（后续版本）
- [ ] 学习资料（后续版本）

### 题库策略

- **AI 动态生成**：基于岗位类型 + 题型，由 ZhipuAI 实时生成题目
- **优势**：无限题目、贴合岗位、避免重复
- **备选**：如需稳定性，可预生成 100 道种子题目入库

## Why This Approach

### 技术选型理由

| 层级 | 选择 | 理由 |
|------|------|------|
| **架构** | Monorepo 全栈 | 单仓库管理，开发效率高，前后端协同方便 |
| **前端** | React + Vite + Tailwind CSS | 现代化开发体验，完美还原 UI 设计 |
| **后端** | FastAPI (Python) | 与 AI 服务（ZhipuAI）无缝集成，异步支持好 |
| **AI 服务** | ZhipuAI GLM-4-Flash | 国内访问稳定，性价比高，PRD 推荐 |
| **语音识别** | 讯飞语音 | 用户指定，国内服务稳定 |
| **部署** | Vercel (前端) + Railway (后端) | 国内访问友好，免费额度，部署简单 |
| **数据存储** | Session Storage（无数据库） | PRD 要求"阅后即焚"，最简方案 |
| **认证** | 无需认证 | 打开即用，降低使用门槛 |

## Key Decisions

### 1. 前后端分离 + Monorepo

```
miansshi2/
├── frontend/          # React + Vite 应用
├── backend/           # FastAPI 应用
├── docs/
└── pencil-new.pen     # UI 设计源文件
```

### 2. PC 端和移动端独立实现

**决定**: 基于路由检测设备类型，加载不同的页面组件

```typescript
// 路由策略
const isMobile = /iPhone|iPad|Android/i.test(navigator.userAgent);

// PC 路由
/pc/training      // 训练模式
/pc/exam          // 模拟考试

// 移动端路由
/mobile/training  // 训练模式
/mobile/exam      // 模拟考试
/mobile/feedback  // AI 反馈页
```

### 3. 核心数据流

```
用户作答流程:
┌─────────┐    ┌─────────┐    ┌─────────┐    ┌─────────┐
│ 选择岗位 │ -> │ 获取题目 │ -> │ 录音/文本 │ -> │ AI 评分 │
└─────────┘    └─────────┘    └─────────┘    └─────────┘
     │              │              │              │
     v              v              v              v
 岗位画像      题型规则       讯飞语音       ZhipuAI
 + 滑块值      + 难度        -> 文本        + Prompt
     │                                           │
     └───────────────┬───────────────────────────┘
                     v
              动态权重计算
                     │
                     v
              评分溯源输出
```

### 4. AI Prompt 策略（复用 1.mianshiAI 设计）

评分维度：逻辑性、原则性、共情力、表达力（权重随岗位动态调整）

输出格式：综合得分 + 评分溯源 + 逻辑诊断 + 深度提升

### 5. 错误处理策略

| 场景 | 处理方式 |
|------|----------|
| ZhipuAI 调用失败 | 重试 3 次 + 友好提示"AI 服务繁忙，请稍后重试" |
| 讯飞语音识别失败 | 提示用户切换到文本输入 |
| 网络超时 | 显示加载状态 + 超时后提供重试按钮 |
| 题目生成失败 | 使用预设备用题目 |
