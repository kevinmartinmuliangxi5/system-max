# 双脑协同工作流 - 完整指南

**实现您的原始构想：User → Brain → Dealer → Worker**

---

## 🎯 设计理念

### 原始构想（增强版 v2.0）

```
User（自然语言）
  ↓ "我想实现用户登录功能"
Brain（理解+规划）
  ├─ 🎨 UI 设计选型（如有界面）← 新增！
  │    ├─ 使用 Pencil MCP 生成 3 种风格
  │    └─ 用户选择后存入设计规范
  ├─ 问问题澄清需求
  ├─ 复述任务确认
  ├─ 自动生成蓝图
  └─ 部署给 Dealer
  ↓
Dealer（生成指令）
  ├─ 读取蓝图
  ├─ 检索相关经验（海马体）← 智能检索
  ├─ 生成完整指令
  └─ 复制到剪贴板
  ↓
Worker（执行实现）
  ├─ 接收指令 + 历史经验
  ├─ 理解上下文
  ├─ 编写代码
  └─ 完成任务
  ↓
Brain（Phase 间审查）← 关键！
  ├─ 扫描产出文件
  ├─ 抽检代码质量
  ├─ 📚 存储经验到海马体 ← 强制！
  └─ 推进下一 Phase
```

**✅ 系统会越来越智能！** 海马体不断积累经验，Worker 避免重复犯错。

---

## 🚀 三种实现方式

### 方式 1: 智能 Brain（Claude Code 直接对话）⭐ 推荐

**适用场景**: 使用 Cursor 或支持 Claude Code 的 IDE

**工作流程**:

#### Step 1: 启动 Brain 模式

在 Claude Code 中说：

```
你现在是 Brain，帮我规划任务
```

或者直接粘贴 `brain_prompt.md` 的内容给 Claude Code。

#### Step 2: 自然语言描述任务

```
💬 你: 我想实现用户登录功能，支持邮箱和手机号登录
```

#### Step 3: Brain 问问题澄清

```
🧠 Brain: 好的！让我了解一下：

1. 认证方案：使用JWT token还是Session？
2. 密码加密：使用哪种加密方式？（推荐bcrypt）
3. Token过期：希望多长时间？
4. 目标文件：auth.py 和 models/user.py 吗？
```

#### Step 4: 你回答问题

```
💬 你: 用JWT，bcrypt加密，24小时过期，是的这两个文件
```

#### Step 5: Brain 复述确认

```
🧠 Brain: 📋 让我确认一下：

任务名称：实现用户登录功能
详细要求：
  - 支持邮箱和手机号登录
  - 使用JWT token认证
  - bcrypt加密密码
  - Token 24小时过期
目标文件：
  - auth.py（新建）
  - models/user.py（新建）

这样正确吗？
```

#### Step 6: 确认并生成蓝图

```
💬 你: 对的

🧠 Brain: ✅ 正在生成任务蓝图...

[自动创建/更新 .janus/project_state.json]

✅ 蓝图已生成！是否立即生成执行指令？
```

#### Step 7: 调用 Dealer

```
💬 你: 是的

🧠 Brain: 🚀 正在调用 Dealer...

[运行 python dealer_enhanced.py]

✅ 执行指令已复制到剪贴板！

下一步：粘贴给 Worker（另一个终端的 Claude Code）
```

#### Step 8: 粘贴给 Worker 执行

打开新终端，粘贴指令，Worker（Claude Code）自动完成实现。

### 方式 2: 交互式 Brain（brain.py 脚本）

**适用场景**: 命令行环境，或不想每次都"教"Claude Code

**工作流程**:

#### Step 1: 启动 Brain

```bash
python brain.py
```

输出：
```
╔══════════════════════════════════════════════════════════════════╗
║                                                                  ║
║        🧠 Brain - 任务理解与蓝图生成                             ║
║                                                                  ║
╚══════════════════════════════════════════════════════════════════╝

👋 你好！我是 Brain，来帮你规划任务。

💬 你: _
```

#### Step 2: 描述任务

```bash
💬 你: 实现用户登录功能
```

#### Step 3: 回答问题

```
请为这个任务起一个简短的名称：
任务名称: 实现用户登录

需要补充更详细的实现要求吗？
补充说明: 支持邮箱和手机号，使用JWT和bcrypt

这个任务涉及哪些文件？
目标文件: auth.py, models/user.py
```

#### Step 4: 确认任务

```
✅ 任务确认

📌 任务名称: 实现用户登录
📝 实现要求: 实现用户登录功能。支持邮箱和手机号，使用JWT和bcrypt
📁 目标文件: auth.py, models/user.py

这个任务理解正确吗？
  1. 正确，添加到蓝图
  2. 需要修改
  3. 取消

请选择: 1
```

#### Step 5: 部署任务

```
✨ 任务已添加到蓝图！

💬 你: deploy

🚀 准备部署任务

请选择 Dealer 版本：
  1. 增强版（推荐）
  2. 简化版

请选择: 1
```

#### Step 6: 获取指令

```
✅ Dealer 已生成指令并复制到剪贴板！

下一步：粘贴给 Claude Code
```

## 🎬 完整演示

### 场景：新功能开发

#### 1. User 启动对话

```
# 在 Cursor 中打开 Claude Code

```

#### 2. Brain 进入模式

```
Brain: 🧠 Brain 模式已启动！

我来帮你规划任务。请描述你要做什么？
```

#### 3. User 描述需求

```
你: 我需要实现一个用户注册功能，要验证邮箱，发送欢迎邮件
```

#### 4. Brain 主动提问

```
Brain: 好的！让我了解细节：

1. 注册方式：
   - 仅邮箱？还是同时支持手机号？

2. 邮箱验证：
   - 发送验证链接？还是验证码？
   - 验证链接有效期多久？

3. 密码要求：
   - 最小长度？
   - 是否需要复杂度要求？

4. 欢迎邮件：
   - 发送时机？注册成功立即发送？
   - 邮件模板是否已有？

5. 目标文件：
   - 建议：auth.py, models/user.py, email.py
   - 是否需要调整？
```

#### 5. User 回答

```
你:
1. 仅邮箱注册
2. 发验证码，5分钟有效
3. 最少8位，包含字母和数字
4. 验证成功后立即发送，模板我已经准备好了
5. 文件就这三个，没问题
```

#### 6. Brain 复述确认

```
Brain: 📋 任务确认

任务名称：实现用户注册功能
详细要求：
  - 仅支持邮箱注册
  - 发送邮箱验证码（5分钟有效期）
  - 密码要求：最少8位，包含字母和数字
  - 验证成功后自动发送欢迎邮件
  - 使用已有的邮件模板
目标文件：
  - auth.py（修改）
  - models/user.py（修改）
  - email.py（新建或修改）
操作类型：CREATE
依赖：邮件模板文件

这样正确吗？
```

#### 7. User 确认

```
你: 完全正确！
```

#### 8. Brain 生成蓝图

```
Brain: ✅ 正在生成任务蓝图...

[写入 .janus/project_state.json]

{
  "blueprint": [
    {
      "task_name": "实现用户注册功能",
      "instruction": "创建邮箱注册功能，发送验证码（5分钟有效），密码最少8位包含字母数字，验证成功后发送欢迎邮件",
      "target_files": ["auth.py", "models/user.py", "email.py"],
      "status": "PENDING"
    }
  ]

✅ 任务蓝图已生成！

📊 当前蓝图：1个待处理任务

是否立即生成执行指令？
```

#### 9. User 确认部署

```

```

## 🎭 角色定位

### Brain（大脑）- 规划者

**职责**：
- ✅ 理解用户意图
- ✅ 主动提问澄清
- ✅ 生成任务蓝图
- ✅ 调用 Dealer

**特点**：
- 善于对话
- 注重细节
- 主动思考
- 不执行代码

### Dealer（指挥官）- 协调者

**职责**：
- ✅ 读取任务蓝图
- ✅ 检索历史经验
- ✅ 生成完整指令
- ✅ 提供上下文

**特点**：
- 信息整合
- 上下文完整
- 格式规范
- 不执行代码

### Worker（工人）- 执行者

**职责**：
- ✅ 接收执行指令
- ✅ 编写实现代码
- ✅ 创建/修改文件
- ✅ 完成具体任务

**特点**：
- 执行高效
- 代码完整
- 不省略
- 专注实现
