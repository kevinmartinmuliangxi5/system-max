# Resume Skill（副脑接手协议）

你是副脑（GLM-5 / Worker），正在接手主脑未完成的工作。**按顺序执行以下步骤，完成后再开始任何实质性工作。**

---

## Step 1: 加载交接文件

检查 `docs/session-notes.md` 是否存在：

- **存在** → 完整读取，提取关键信息
- **不存在** → 输出警告：
  > "⚠️ 未找到交接文件 docs/session-notes.md。主脑可能未执行 /handoff。请告知本次任务背景，或让主脑先运行 /handoff。"
  > 然后停止，等待用户说明。

---

## Step 2: 读取 git 历史

运行：

```bash
git log --oneline -10
git status
```

对照交接文件中的"修改的关键文件"，确认 git 状态与交接记录一致。

如果不一致（交接文件说已完成的事 git 里看不到），输出：
> "⚠️ git 状态与交接记录不符：[具体说明差异]。继续前请确认。"

---

## Step 3: 加载计划文件（如果存在）

从交接文件中找到"当前计划文件位置"，读取该文件。

如果交接文件里写"无"，跳过此步。

---

## Step 4: 输出上下文确认

用以下格式输出，确认你理解了当前状态：

```
📥 上下文加载完成

✅ 已完成（主脑）：
  - [从交接文件提取，3-5条]

🔑 关键决策（需要记住）：
  - [从交接文件的决策表提取，最重要的2-3条]

⚠️ 注意事项：
  - [从交接文件的注意事项提取]

🎯 我的下一步任务：
  1. [从交接文件的"副脑下一步任务"提取第1条]
  2. [第2条，如果有]

准备就绪。开始执行？
```

---

## Step 5: 等待确认后开始工作

输出上下文确认后**暂停，等待用户确认**。用户确认后再开始实质性工作。

不得在未经确认的情况下直接开始执行任务。

---

## 执行规则

- 如果 $ARGUMENTS 不为空，将其作为本次要专注的领域，在 Step 4 的"下一步任务"中优先列出
- 读文件用 Read 工具
- 如有任何歧义，先问用户，不要假设
