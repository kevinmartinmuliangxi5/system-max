# Handoff Skill（主脑 → 副脑 交接协议）

你正在执行主脑到副脑的切换交接。**按顺序执行以下步骤，不得跳过。**

---

## Step 1: 确认工作状态

运行以下命令，将结果用于后续步骤：

```bash
git status
git log --oneline -8
```

如果有未提交的改动，先提示用户：
> "有未提交的改动，建议先 commit 再交接。是否继续？"

---

## Step 2: 确认计划文件存在（如果本次有规划阶段）

检查以下位置是否存在计划文件：
- `PLAN.md`
- `docs/plan.md`
- `docs/PLAN.md`

如果存在，记录文件路径备用。如果不存在，在交接文件中注明"本次无独立计划文件"。

---

## Step 3: 生成交接文件

将以下内容写入 `docs/session-notes.md`（**覆盖写入，不是追加**）：

```markdown
# 主脑交接记录

> 时间：[当前日期时间]
> 执行脑：主脑（Claude Sonnet / Pro Plan）

---

## 本次完成的工作

[用 3-7 条 bullet 列出，要具体，不要写废话]
-
-

## 关键决策及理由

[这是最重要的部分。列出所有不显而易见的决策，说明为什么这样选择而不是其他方案]

| 决策 | 选择 | 理由 | 备选方案（放弃原因） |
|------|------|------|------------------|
|      |      |      |                  |

## 修改的关键文件

[从 git log/status 中提取]
-

## 副脑下一步任务

[明确、可执行的下一步，副脑接手后第一件事是什么]
1.
2.

## 注意事项 / 陷阱

[副脑需要知道的坑、限制、未解决的问题]
-

## 当前计划文件位置

[如果有：填写路径。如果没有：写"无"]

```

---

## Step 4: 确认交接就绪

输出以下确认信息：

```
✅ 交接文件已写入 docs/session-notes.md
📋 本次工作摘要：[一句话概括]
🔜 副脑下一步：[第一个具体任务]

副脑启动后请运行 /resume 加载上下文。
```

---

## 执行规则

- Step 3 的"关键决策及理由"部分不得留空——这是副脑最需要的信息
- 写文件用 Write 工具，不用 echo 重定向
- 如果 $ARGUMENTS 不为空，将其作为"本次完成的工作"的第一条
