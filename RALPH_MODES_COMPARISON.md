# Ralph 三种使用模式对比

## 模式1：自动模式（适合睡觉时运行）

```bash
bash start_ralph_with_zhipu.sh --live --verbose
```

### 特点
- ✅ 完全自动，无需人工干预
- ✅ 持续运行直到完成
- ✅ 适合夜间运行
- ❌ 无法中途修改
- ❌ 可能走弯路

### 适用场景
- 明确的任务需求
- 不需要频繁调整
- 长时间运行（过夜）

### 详细程度：★★★☆☆

---

## 模式2：监控模式（Windows Terminal三分屏）

```bash
# 面板1: Brain对话
cd D:/AI_Projects/system-max

# 面板2: Ralph执行
bash start_ralph_with_zhipu.sh --live --verbose

# 面板3: 实时监控
bash monitor_ralph.sh
```

### 特点
- ✅ 实时查看3个视图
- ✅ 可以看到详细日志
- ✅ 可以监控状态和成本
- ⚠️ 需要手动干预才能修改

### 适用场景
- 需要监控进度
- 想看到实时状态
- 中等复杂度任务

### 详细程度：★★★★★

---

## 模式3：交互式模式（推荐用于MVP测试）⭐

```bash
bash ralph_interactive.sh
```

### 特点
- ✅ 每轮后暂停，等待您的指令
- ✅ 可以随时提供反馈
- ✅ 可以查看代码差异
- ✅ 可以修改Worker的行为
- ✅ 完全可控

### 适用场景
- **MVP测试阶段（强烈推荐）**
- 学习Ralph工作流程
- 需要频繁调整的任务
- 关键项目（需要把关）

### 详细程度：★★★★★★

### 交互命令

每轮执行后，您可以输入：

| 命令 | 说明 | 示例 |
|------|------|------|
| `c` / `continue` | 继续下一轮 | - |
| `f` / `feedback` | 提供反馈意见 | "这里用UUID而不是自增ID" |
| `d` / `diff` | 查看代码差异 | - |
| `v` / `view` | 查看完整输出 | - |
| `s` / `skip` | 跳过当前任务 | - |
| `q` / `quit` | 退出Ralph | - |
