# 🚀 快速开始指南

---

## ⚡ 1分钟快速启动

### 步骤1: 安装依赖
```bash
pip install openai python-dotenv
```

### 步骤2: 配置API密钥
```bash
# Windows PowerShell
$env:ZHIPUAI_API_KEY="你的API密钥"

# Mac/Linux
export ZHIPUAI_API_KEY="你的API密钥"
```

### 步骤3: 运行
```bash
python agent_discussion_system.py
```

完成！观看5个AI角色讨论你的任务 🎉

### 正常输出示例
```
============================================================

============================================================

============================================================
📢 第 1 轮讨论

✨ [涌现时刻] 检测到新想法或突破性观点！
```

### 涌现时刻标识
- ✨ = 检测到新想法、优化建议、突破性观点
- 这是系统认为的"协作价值"产生的时刻

### 最终报告
```
📊 讨论报告
============================================================
🎯 任务：...
📝 轮次：3
💬 发言：15 条
✨ 涌现时刻：2 次

📋 总结：...
```

## 🐛 常见问题

### Q1: API调用失败
```
错误: API调用失败
```

**解决**:
1. 检查API密钥是否正确
2. 检查网络连接
3. 确认已订阅GLM Coding Plan

### Q2: 输出内容质量不高
**解决**:
1. 增加讨论轮次（如5轮）
2. 调整温度参数（0.7-0.9）
3. 优化Agent的system_prompt

### Q3: 响应太慢
**解决**:
1. 减少讨论轮次
2. 使用GLM-4.5-Air模型（更快）
3. 减少Agent数量

### Q4: 想加入人类参与讨论
**未来功能**: 计划支持人类作为特殊Agent加入讨论

## 💡 最佳实践

### 1. 任务描述要清晰
❌ 差: "写个排序"
✅ 好: "实现快速排序算法，要求时间复杂度O(nlogn)"

### 2. 适当的讨论轮次
- 简单任务: 2-3轮
- 中等任务: 3-4轮
- 复杂任务: 5-6轮

### 3. 观察涌现时刻
注意✨标记出现的时机，这些通常是最有价值的观点

### 4. 阅读总结
最终总结会提炼出协作的价值，值得仔细阅读

## 🎓 深入理解

### 这个系统为什么是"真涌现"？

```
传统遗传算法:
代码A + 代码B → 机械变异 → 代码C
         ↑
      无智能理解

多Agent讨论:
Architect想法 + Coder实现 + Critic质疑 + Optimizer改进
         ↓
     智能协作、思想碰撞
         ↓
    最终方案 > 任何单个Agent
         ↑
      这就是涌现！✨
```

### 关键特征

1. **语义理解**: Agents真正理解任务
2. **智能协作**: 不是机械组合，是智能互补
3. **1+1>2**: 协作产生的价值大于个体之和
4. **自然语言**: 讨论过程完全可读
