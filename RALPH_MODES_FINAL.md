## ⭐ **强烈推荐：自动流式输出模式**

### **启动命令：**

```bash
cd D:/AI_Projects/system-max
bash ralph_auto_stream.sh
```

### **工作原理：**

```
Loop 1: Worker实时流式输出 → 每秒显示进度 → 倒计时5秒
        ↓
        您可以按 Ctrl+C 打断 → 进入交互菜单
        或者
        5秒后自动继续 ↓

Loop 2: Worker实时流式输出 → 看到每一步 → 倒计时5秒
        ↓
        您按了 Ctrl+C！→ 暂停 → 提供反馈 → 继续
        ↓

Loop 3: Worker根据反馈修正 → 实时显示 → 倒计时5秒
        ↓
        没有打断 → 自动继续
        ↓

Loop 4: Worker完成 → <promise>COMPLETE</promise> → 自动退出
```

### **特点：**
- ✅ 自动连续迭代（无需每次输入c）
- ✅ **实时流式输出**（像 `--live --verbose` 一样每秒更新）
- ✅ 每轮结束后5秒倒计时
- ✅ 倒计时期间按 **Ctrl+C** 可打断
- ✅ 打断后进入交互菜单（**交互稳定，打字正常**）
- ✅ 可提供反馈、查看diff、查看输出
- ✅ 继续后恢复自动模式

## 🚨 **重要提醒**

### **如果您正在使用 ralph_auto_with_interrupt.sh：**

```bash
# 立即停止
按 Ctrl+C 多次强制退出

# 使用正确的脚本
cd D:/AI_Projects/system-max
bash ralph_auto_stream.sh
```

### **为什么 ralph_auto_with_interrupt.sh 有问题？**

1. **打字无反应：** 使用 `stty -echo -icanon` 修改终端设置，在 Git Bash 环境下会导致输入失效
2. **没有实时输出：** 将Worker输出存储到变量，只显示前30行，看不到实时进度
3. **按键检测不稳定：** "按任意键"机制在某些终端不工作

### **ralph_auto_stream.sh 如何解决？**

1. **交互稳定：** 使用标准 Ctrl+C 信号处理，不修改终端设置，交互完全正常
2. **实时流式输出：** 使用 `tee` 命令同步显示和保存，像 `--live --verbose` 一样每秒更新
3. **可靠的打断：** Ctrl+C 是标准信号，在所有终端环境都可靠工作
