# Ralph 自动流式输出模式 - 使用指南

## ✅ 新脚本：ralph_auto_stream.sh

**解决的问题：**
1. ✅ 实时流式输出 - 像 `--live --verbose` 一样每秒显示Worker动态
2. ✅ 自动迭代 - 无需输入c，自动连续运行
3. ✅ 随时可打断 - 按 **Ctrl+C** 打断（不是按任意键）
4. ✅ 可提供反馈 - 打断后进入交互菜单

---

```bash
cd D:/AI_Projects/system-max
bash ralph_auto_stream.sh
```

## 🔑 关键区别

### ❌ ralph_auto_with_interrupt.sh（有问题）

```
问题1：没有实时流式输出
  - Worker执行时只显示"执行中..."
  - 看不到Worker每秒在做什么
  - 只能看结果的前30行

问题2：交互有bug
  - 使用 stty 改变终端设置
  - 在某些终端（Git Bash）可能导致打字无响应
  - 按任意键检测机制不稳定
```

### ✅ ralph_auto_stream.sh（推荐）

```
优点1：完整实时流式输出
  - 使用 tee 命令同步显示和保存
  - 看到Worker的每一个输出
  - 像 --live --verbose 一样的体验

优点2：可靠的交互
  - 使用标准 Ctrl+C 信号
  - 不修改终端设置
  - 交互菜单完全正常工作
```

## 🚨 常见问题

### Q1：看不到实时输出？

**确认使用的是 `ralph_auto_stream.sh`，不是 `ralph_auto_with_interrupt.sh`**

```bash
# 正确的命令
bash ralph_auto_stream.sh

# 错误的命令（旧版本）
bash ralph_auto_with_interrupt.sh
```

---

### Q2：按 Ctrl+C 后打字没反应？

**那说明用错脚本了！应该用 `ralph_auto_stream.sh`**

```bash
# 停止当前脚本
Ctrl+C （多按几次）

# 使用正确的脚本
bash ralph_auto_stream.sh
```

---

### Q3：想完全停止Ralph怎么办？

```bash
# 方法1：在倒计时期间按 Ctrl+C
⏱️  5秒后自动继续...
[按 Ctrl+C]

👉 您的指令: q

# 方法2：在任何时候多次按 Ctrl+C 强制退出
```

## 💡 最佳实践

### 1. 先观察2-3轮

```
Loop 1: 不打断，看Worker能做到什么程度
Loop 2: 不打断，看Worker是否继续正确
Loop 3: 发现问题 → 按 Ctrl+C → 提供反馈
```
